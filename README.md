1. 순서도 작성
   1) 프로그램의 절차 수는 정해져 있어야 한다.
   2) 각 절차는 항상 같은 내용이어야 한다.
   3) 모든 가능성을 고려해야 한다.
   4) 예는 절차를 검증할 떄 사용한다.
   *사용자의 이벤트(버튼 클릭, 입력창 글자 입력 등)가 필요한 곳에서 순서도를 끊어야 함!
